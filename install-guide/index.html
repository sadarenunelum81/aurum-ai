AurumAI Installation Guide

A complete step-by-step guide to deploying and configuring your AI Auto Blogger platform.

Overview

This guide will walk you through deploying your application using Vercel, setting up the necessary Firebase services (Authentication and Firestore), and configuring your application with the required API keys through the admin panel.

The process is broken down into four main parts:

  1. Deploying the Project to Vercel
  2. Setting up Your Firebase Project
  3. Getting Your API Keys (Firebase & Gemini)
  4. Configuring the Application via the Admin Panel

Part 1: Deploying to Vercel

Vercel is the recommended hosting platform for Next.js projects, offering seamless deployment and a generous free tier.

  1. Push to GitHub: If you haven't already, create a new repository on GitHub and push the project code to it.
  2. Sign up for Vercel: Create a free account at vercel.com using your GitHub account.
  3. Import Project: From your Vercel dashboard, click "Add New..." → "Project".
  4. Connect Git Repository: Find and import the GitHub repository you just created. Vercel will automatically detect that it's a Next.js project.
  5. Deploy: You do not need to configure any environment variables at this stage. Simply click the "Deploy" button. Vercel will build and deploy your site.
  6. Get Your URL: Once deployed, Vercel will assign you a public URL (e.g., your-project.vercel.app). Keep this URL handy; you will need it later.
Note: We will configure API keys and secrets in a later step through the application's admin panel, not in Vercel's environment variables dashboard.

Part 2: Setting up Your Firebase Project

Firebase will handle your user authentication and database needs.

Step 2.1: Create a Firebase Project

  1. Go to the Firebase Console and click "Add project".
  2. Give your project a name (e.g., "My AI Blog") and click "Continue".
  3. You can choose to enable or disable Google Analytics. For this guide, you can disable it. Click "Create project".
  4. Wait for your project to be created.

Step 2.2: Enable Firestore Database

  1. In your new project's console, navigate to the "Build" section in the left sidebar and click on "Firestore Database".
  2. Click the "Create database" button.
  3. Select "Start in production mode" and click "Next".
  4. Choose a Firestore location closest to your target audience. Click "Enable".

Step 2.3: Enable Firebase Authentication

  1. In the "Build" section of the sidebar, click on "Authentication".
  2. Click the "Get started" button.
  3. In the "Sign-in method" tab, you need to enable at least one provider. For this application, select **Email/Password**.
  4. Enable the "Email/Password" provider and click "Save".
Your Firebase project is now ready. Next, we will get the credentials needed to connect it to your application.

Part 3: Getting Your API Keys

Your application needs two sets of keys: one to connect to Firebase, and one to use Google's AI for content generation.

Step 3.1: Get Your Firebase Configuration

  1. In your Firebase Console, click the **Project Overview** gear icon and select "Project settings".
  2. Under the "Your apps" section, click the web icon () to create a new web app.
  3. Give your app a nickname (e.g., "Web App") and click "Register app".
  4. On the next screen, "Add Firebase SDK", you will see a section with `firebaseConfig`. **You do not need to copy this code block.** The application is already set up to handle it. You will connect it through the admin panel in the next step. Simply click "Continue to console".

Step 3.2: Get Your Gemini API Key

  1. Go to Google AI Studio.
  2. Click "Create API key in new project".
  3. A new API key will be generated for you. Copy this key and save it somewhere safe.

Part 4: Configuring the Application

The final step is to input all the credentials into your application's admin panel.

  1. Access the Admin Panel: Go to your deployed Vercel URL and add /admin/login to the end (e.g., https://your-project.vercel.app/admin/login).
  2. Create Your Admin Account: Since this is the first time you are accessing the site, the first account you create will automatically be assigned the **admin** role. Use your own email and a secure password to sign up.
  3. Navigate to Settings: After logging in, you will be redirected to the admin dashboard. In the sidebar, navigate to the **Auto Blogger** page.
  4. Enter Your Project URL: In the "API Key & URL Configuration" card, enter the full Vercel URL from Part 1 (e.g., https://your-project.vercel.app) into the "Project URL" field.
  5. Enter Your Gemini API Key: Paste the Gemini API key you got from Part 3.2 into the "Google Gemini API Key" field.
  6. Save Settings: Click the "Save Settings" button.
Important: You do not need to manually enter the Firebase config. The application is built to handle this securely. As long as your Firebase Project ID is correctly set up in the backend, the connection will work automatically. By deploying to Vercel and setting up the project, the system handles the Firebase connection for you.

Setup Complete!

Your application is now fully deployed and configured. You can now explore the admin panel to set up the Auto Blogger, customize templates, and manage your content.
